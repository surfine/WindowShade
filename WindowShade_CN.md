# WindowShade 设计笔记

[English](WindowShade.md)

WindowShade 来自 classic Mac OS 里的一个窗口动作：双击标题栏，窗口内容向上收起，只留下标题栏；再双击一次，窗口从原处展开。

这个动作看起来很小，细节却很要紧。窗口没有离开桌面，没有进 Dock，也没有被放到另一个列表里。入口还在你刚才看见的位置。

WindowShade 这个小工具做的，就是用今天 macOS 还能给第三方应用使用的 API，把这种手感重新做一遍。

## 为什么做它

macOS 已经有很多窗口管理方式。可以最小化到 Dock，可以用 Mission Control 找窗口，可以开 Stage Manager，也可以把窗口平铺。

这些方式都有效，只是经常会把“找回窗口”的位置换掉。

WindowShade 处理的是更小的场景：一个窗口挡住了后面的东西，你只是想看一眼。不想关闭它，不想隐藏整个 app，也不想把它丢进 Dock。

于是它先卷起来。

## 一点历史

WindowShade 一开始并不是 Apple 的系统功能。早期版本通常归功于 Rob Johnston，他为 classic Mac OS 写了这个第三方工具。Apple 后来买下权利，并在 1994 年的 System 7.5 里把它带进系统。

到了 Mac OS X，系统内建的 WindowShade 消失了。窗口管理转向 Dock 最小化、Expose 和后来的 Mission Control。之后还有 WindowShade X、WindowMizer、Deskovery 这样的第三方工具，用不同方式把这个想法留了下来。

今天的 macOS 里还能看到一点第一方遗存：Stickies。便笺可以折成彩色小条，位置和身份都还在。它不像一个被最小化的窗口，更像一张被临时卷起的纸。

我想保留的就是这一点。

## 现在这版怎么做

现代 macOS 不允许一个 app 直接重画另一个 app 的窗口。WindowShade 只能绕开一点做。

现在的流程大概是：

1. 用 Accessibility API 找到当前窗口。
2. 用 ScreenCaptureKit 截取窗口顶部。
3. 在原位置生成一条 AppKit 覆盖层，作为卷帘条。
4. 把真实窗口移到屏幕外、隐藏，或最小化。
5. 用户展开时，再把真实窗口带回来。

主代码在 `prototype/WindowShade.swift`。

可见样式有两种：

- 原貌卷帘：保留真实窗口顶部截图。
- 标准标题栏：用统一的代理标题栏表示。

已折叠的窗口可以从卷帘条、菜单栏列表、数字快捷键找回。专注 shelf 会把其他 app 的窗口收到屏幕顶部，让当前 app 留在前面，同时不把整个桌面切进另一套管理模式。

这些做法都有取舍。WindowShade 更关心那个操作含义：窗口还在这里，只是内容先收起来。

也可以把它看成“展开”和“离开桌面”之间的一个中间状态。内容退后，标题、位置和返回入口留下。

## 现在的限制

有些 app 很好处理，有些 app 不太配合。

自绘标题栏、全屏空间、Stage Manager、多显示器，以及带很多浮动面板的专业软件，都可能需要单独适配。Stickies 本身已经有原生折叠行为，WindowShade 应该尽量让路。

这版也会碰到一些私有或半私有的系统行为。它适合拿来试用和研究，还不能当成非常稳定的发行版。

## 一个设计原则

如果一个窗口只是暂时挡路，找回它的入口就应该留在原地。
